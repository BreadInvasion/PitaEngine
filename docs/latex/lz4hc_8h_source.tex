\DoxyCodeLine{112 \textcolor{comment}{/*}}
\DoxyCodeLine{113 \textcolor{comment}{  These functions compress data in successive blocks of any size,}}
\DoxyCodeLine{114 \textcolor{comment}{  using previous blocks as dictionary, to improve compression ratio.}}
\DoxyCodeLine{115 \textcolor{comment}{  One key assumption is that previous blocks (up to 64 KB) remain read-\/accessible while compressing next blocks.}}
\DoxyCodeLine{116 \textcolor{comment}{  There is an exception for ring buffers, which can be smaller than 64 KB.}}
\DoxyCodeLine{117 \textcolor{comment}{  Ring-\/buffer scenario is automatically detected and handled within LZ4\_compress\_HC\_continue().}}
\DoxyCodeLine{118 \textcolor{comment}{}}
\DoxyCodeLine{119 \textcolor{comment}{  Before starting compression, state must be allocated and properly initialized.}}
\DoxyCodeLine{120 \textcolor{comment}{  LZ4\_createStreamHC() does both, though compression level is set to LZ4HC\_CLEVEL\_DEFAULT.}}
\DoxyCodeLine{121 \textcolor{comment}{}}
\DoxyCodeLine{122 \textcolor{comment}{  Selecting the compression level can be done with LZ4\_resetStreamHC\_fast() (starts a new stream)}}
\DoxyCodeLine{123 \textcolor{comment}{  or LZ4\_setCompressionLevel() (anytime, between blocks in the same stream) (experimental).}}
\DoxyCodeLine{124 \textcolor{comment}{  LZ4\_resetStreamHC\_fast() only works on states which have been properly initialized at least once,}}
\DoxyCodeLine{125 \textcolor{comment}{  which is automatically the case when state is created using LZ4\_createStreamHC().}}
\DoxyCodeLine{126 \textcolor{comment}{}}
\DoxyCodeLine{127 \textcolor{comment}{  After reset, a first "{}fictional block"{} can be designated as initial dictionary,}}
\DoxyCodeLine{128 \textcolor{comment}{  using LZ4\_loadDictHC() (Optional).}}
\DoxyCodeLine{129 \textcolor{comment}{}}
\DoxyCodeLine{130 \textcolor{comment}{  Invoke LZ4\_compress\_HC\_continue() to compress each successive block.}}
\DoxyCodeLine{131 \textcolor{comment}{  The number of blocks is unlimited.}}
\DoxyCodeLine{132 \textcolor{comment}{  Previous input blocks, including initial dictionary when present,}}
\DoxyCodeLine{133 \textcolor{comment}{  must remain accessible and unmodified during compression.}}
\DoxyCodeLine{134 \textcolor{comment}{}}
\DoxyCodeLine{135 \textcolor{comment}{  It's allowed to update compression level anytime between blocks,}}
\DoxyCodeLine{136 \textcolor{comment}{  using LZ4\_setCompressionLevel() (experimental).}}
\DoxyCodeLine{137 \textcolor{comment}{}}
\DoxyCodeLine{138 \textcolor{comment}{  'dst' buffer should be sized to handle worst case scenarios}}
\DoxyCodeLine{139 \textcolor{comment}{  (see LZ4\_compressBound(), it ensures compression success).}}
\DoxyCodeLine{140 \textcolor{comment}{  In case of failure, the API does not guarantee recovery,}}
\DoxyCodeLine{141 \textcolor{comment}{  so the state \_must\_ be reset.}}
\DoxyCodeLine{142 \textcolor{comment}{  To ensure compression success}}
\DoxyCodeLine{143 \textcolor{comment}{  whenever `dst` buffer size cannot be made >= LZ4\_compressBound(),}}
\DoxyCodeLine{144 \textcolor{comment}{  consider using LZ4\_compress\_HC\_continue\_destSize().}}
\DoxyCodeLine{145 \textcolor{comment}{}}
\DoxyCodeLine{146 \textcolor{comment}{  Whenever previous input blocks can't be preserved unmodified in-\/place during compression of next blocks,}}
\DoxyCodeLine{147 \textcolor{comment}{  it's possible to copy the last blocks into a more stable memory space, using LZ4\_saveDictHC().}}
\DoxyCodeLine{148 \textcolor{comment}{  Return value of LZ4\_saveDictHC() is the size of dictionary effectively saved into 'safeBuffer' (<= 64 KB)}}
\DoxyCodeLine{149 \textcolor{comment}{}}
\DoxyCodeLine{150 \textcolor{comment}{  After completing a streaming compression,}}
\DoxyCodeLine{151 \textcolor{comment}{  it's possible to start a new stream of blocks, using the same LZ4\_streamHC\_t state,}}
\DoxyCodeLine{152 \textcolor{comment}{  just by resetting it, using LZ4\_resetStreamHC\_fast().}}
\DoxyCodeLine{153 \textcolor{comment}{*/}}
